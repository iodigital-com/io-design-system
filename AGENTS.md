# io Design System — Agent Instructions

Web Component library for **iO Digital**. Stencil.js core with auto-generated React, Vue, and Angular wrappers.

## Build & Test

Run from **repo root** (`io-design-system/`). Node >= 20 required.

```bash
npm ci                        # Install dependencies
npm run dev                   # Stencil watch + storefront at localhost:3000
npm run build                 # Build core + all framework wrappers
npm run test                  # Vitest unit tests
npm run type-check            # TypeScript check (storefront only)
npm run governance:check      # Validate workspace invariants — run before every commit
npm run events:guard          # Guard against io-prefixed custom event names
npm run build:quality-gates   # Full CI: governance → events:guard → build → test → type-check → storefront
npm run agents:install:claude # Install curated Claude agents from agency-agents registry
```

Build granularity:

```bash
npm run build:components      # Stencil core only
npm run build:wrappers        # React + Vue + Angular wrappers only
npm run build:storefront      # Next.js docs site only
```

## Architecture

```
io-components/          @io-digital/components — Stencil core (edit here)
  src/components/       One directory per component (io-button/, io-modal/, …)
  src/global/app.css    Design tokens as CSS custom properties on :root
  src/global/app.ts     Global script — calls initFocusVisible()
  src/utils/            Shared utilities (focus-visible.ts)
io-storefront/          Private Next.js documentation site
  src/app/components/   Component tab pages (5 tabs per component)
  src/components/       Shared UI (layout, playground, configurator)
  src/utils/generator/  Code-generation utilities + IoTagNames union
  src/types/            custom-elements.d.ts — JSX typings
  src/sitemap.ts        Navigation tree (add new components here)
io-components-react/    Auto-generated wrappers — NEVER edit manually
io-components-vue/      Auto-generated wrappers — NEVER edit manually
io-components-angular/  Auto-generated wrappers — NEVER edit manually
docs/agency-agents/     AI governance: curated-io-design-system.json + ADAPTATION_LAYER.md
```

Wrapper packages are regenerated by `npm run build:components`. See [CONTRIBUTING.md](CONTRIBUTING.md).

## Component File Layout

Every component lives at `io-components/src/components/io-{name}/`:

| File | Purpose |
|---|---|
| `io-{name}.tsx` | Stencil component class |
| `types.ts` | Exported prop type unions (`IoButtonVariant`, etc.) |
| `io-{name}-styles.ts` | `getButtonStyles()` / CSS-in-JS injected into Shadow DOM |
| `io-{name}-utils.ts` | Pure logic, ARIA builders — unit-tested separately |
| `io-{name}.spec.ts` | Default props / render tests |
| `io-{name}.click.spec.ts` | Event-emission tests |
| `io-{name}.disabled.spec.ts` | Disabled-state tests |

## Conventions

**Styling — tokens only:**
- Never hardcode hex values, pixel sizes, or radii — always use `var(--io-*)` CSS custom properties.
- Token hierarchy: `--io-{category}-{scale}` (primitive) → `--io-{role}` (semantic) → `--io-{component}-{prop}` (consumer API).
- All tokens are defined in `io-components/src/global/app.css`.
- New design values: add to `app.css` first, then reference via `var(--io-*)`.
- Import styles from `-styles.ts`; never write CSS inline in `.tsx`.

**Shadow DOM:** All components use `shadow: { delegatesFocus: true }`. CSS custom properties are the only styling API that crosses the boundary.

**Focus rings:** Use `var(--io-focus-ring-active)` — never `var(--io-shadow-focus-ring)` directly. The `initFocusVisible()` utility in `src/global/app.ts` sets this custom property based on keyboard vs. pointer input modality.

**IDs:** Generate IDs in `componentWillLoad()`, not in `render()`.

**Events:** Do not add `io`-prefixed custom event names. Verify with `npm run events:guard`.

**Naming:**
- Web Components: `io-{name}` (kebab-case tag, `Io{Name}` Stencil class)
- Framework wrappers: `Io{Name}` (PascalCase)
- Type unions: `Io{Name}{Prop}` (e.g. `IoButtonVariant`)

**TypeScript:** strict mode. `@ts-expect-error` is not permitted as a workaround — add correct types to `custom-elements.d.ts` instead.

**Tests (Vitest + jsdom):**
- `vi.mock()` must be at the top level of the spec file, not inside `beforeEach`.
- Every component requires: render test, event-emission test, disabled-state test.
- WCAG AA minimum: 4.5:1 text contrast, 3:1 UI component contrast.
- Interactive elements: 44 × 44 px minimum touch target.

## Adding a New Component

1. Create `io-components/src/components/io-{name}/` with all required files (see layout above).
2. Create `io-storefront/src/app/components/io-{name}/` with all five tab pages (`layout.tsx`, `page.tsx`, `configurator/`, `examples/`, `usage/`, `accessibility/`, `api/`, `io-{name}.stories.ts`).
3. Register the tag in **two files** (both or TypeScript errors):
   - `io-storefront/src/utils/generator/generator.tsx` → add to `IoTagNames` union
   - `io-storefront/src/types/custom-elements.d.ts` → add JSX interface
4. Add to `io-storefront/src/sitemap.ts` in alphabetical order (only after all five tab pages exist).

## Storefront Shared Primitives

- Usage pages: import from `io-storefront/src/components/usage/UsagePrimitives.tsx` — do not redefine `SectionHeader`, `RuleCard`, or `DoOrDont` locally.
- Accessibility pages: import from `io-storefront/src/components/accessibility/AccessibilityPrimitives.tsx`.
- Component relations: defined in `io-storefront/src/sitemap.ts` (`slug`, `description`, `related`), rendered via `io-storefront/src/components/RelatedComponents.tsx` in all `io-*/layout.tsx` files.

## Quality Gates

Run before any commit — especially for UI/component changes:

```bash
npm run governance:check   # Required — validates workspace topology and governance files
npm run events:guard       # Required — no io-prefixed custom events
npm run build
npm run test
npm run type-check
npm run build:storefront
```

Or run all in sequence: `npm run build:quality-gates`

## Commit Messages

Use [Conventional Commits](https://www.conventionalcommits.org/): `<type>(<scope>): <subject>`

| Type | When |
|---|---|
| `feat` | New component or feature |
| `fix` | Bug fix |
| `refactor` | Code change that is not a fix or feature |
| `docs` | Documentation only |
| `test` | Adding or updating tests |
| `chore` | Build scripts, dependency updates, tooling |

Example: `feat(io-button): add xl size variant`

## Do Not Commit

The following are local-only and must never be committed:

- `design-system/`
- `prompts/`
- `issues/`
- `.codex/`
